Help & Documentation
The Pipeline Workflow
MediaManager follows a 4-step process:
  • 0. Scan: Discovering video files on your network drive.
  • 1. Identify: Auto-matching via TMDB/OMDB (Fuzzy search).
  • 2. LLM: Sending tricky filenames to an AI for deep analysis.
  • 3. Apply: The final "commit" that renames and moves files.
Manual Review Tips
Need to override a match?
  • Go to the Library and click any file.
  • The side panel lets you search TMDB manually.
  • Select the correct match and click Verify Result.
  • Set status to Identified to mark it ready for applying.
Local AI Setup (Ollama)
To use local identification:
  • Install Ollama or LM Studio.
  • Ensure the server is running on your machine.
  • Go to Settings → LLM and pick the provider.
  • Select a model (like Llama3 or Mistral) and hit Save.
Safety & Rollback
Don't worry about breaking things:
  • Always run Dry Run first to see the changes.
  • The Verify & Apply page shows a before/after view.
  • If you make a mistake, use Rollback in the pipeline tab.
  • Original files are only deleted after a successful copy-verify.
